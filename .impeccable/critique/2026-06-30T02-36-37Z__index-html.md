The main product-trust issue is inconsistency: a local-first app says `Free plan · Upgrade`, mixes English/Afrikaans unevenly, exposes advanced terms early, and uses non-semantic clickable cards.

## Overall Impression

Flora already has the bones of a serious local AI tool: focused, restrained, and genuinely capable. The biggest opportunity is to make local state, controls, and copy feel intentionally designed instead of assembled around a chatbot shell.

## What's Working

- The centered composer creates a calm, high-focus starting point.
- The feature set is materially useful: files, web grounding, image/video routes, model routing, voice, export, and sandbox previews all support real work.
- The restrained dark palette avoids the loud gradient-chatbot pattern and fits a local power-user tool.

## Priority Issues

**[P1] Local trust is undermined by SaaS-style status copy**  
**Why it matters**: A local-first product loses credibility when the header says `Free plan · Ollama · Upgrade`; users need to trust local service state before prompting.  
**Fix**: Replace pseudo-content on `.status-pill` with real service status: checking, connected, offline, active model, and optional Wan state. Make diagnostics available from that status.  
**Suggested command**: `$impeccable clarify index.html`

**[P1] Mobile first screen clips and shrinks important controls**  
**Why it matters**: On phone-sized viewports, suggestion chips and small 26-30px controls make the first action feel fragile.  
**Fix**: Rework the mobile suggestion layout, remove clipping, and raise interactive targets to at least 44px while keeping the composer in the thumb zone.  
**Suggested command**: `$impeccable adapt index.html`

**[P1] Accessibility semantics are not strong enough for a tool**  
**Why it matters**: Clickable `div` cards, weak focus states, tiny text, and visual-only state cues exclude keyboard and assistive-tech users.  
**Fix**: Convert suggestion cards, nav/history actions, and session controls to real buttons or links; add `:focus-visible`; use `aria-live` for loading/errors; improve placeholder and tiny-text contrast.  
**Suggested command**: `$impeccable audit index.html`

**[P2] Tools menu exposes too many concepts at once**  
**Why it matters**: Grounding, Google grounding, temperature, Wan endpoint, workflow JSON, save/test/clear, and export all appear in one dense panel before the user has succeeded once.  
**Fix**: Split into Capabilities, Tuning, and Video; hide raw Wan JSON behind Advanced; show configured/unconfigured state clearly.  
**Suggested command**: `$impeccable layout index.html`

**[P2] Long-running generation needs an escape hatch**  
**Why it matters**: Local model and video work can hang; disabling the interface without a stop path creates anxiety and refresh-risk.  
**Fix**: Add a Stop button with `AbortController`, preserve the prompt, and keep non-conflicting controls usable during generation.  
**Suggested command**: `$impeccable harden index.html`

## Persona Red Flags

**Alex (Power User)**: Finds shortcuts and drag/drop quickly, but cannot stop a long Ollama or Wan run. Automatic model switching is useful, but the active route and fallback behavior are not explained.

**Jordan (First-Timer)**: Encounters `Grounding`, `Google Grounding`, `Wan 2.2`, model IDs, and mixed language before learning the product. The path from local setup to first successful prompt is not clearly taught.

**Sam (Accessibility-Dependent User)**: Several controls are not keyboard-native, focus styling is thin, text can be as small as 9px, and status is partly visual or pseudo-content.

**Casey (Mobile User)**: Mobile layout risks clipped suggestion cards, small touch targets, and session loss on interruption despite the interface implying history.
